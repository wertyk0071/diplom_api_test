## Описание проекта
Проект содержит автотест для проверки API сервиса заказа самоката. Тест написан на Python с использованием библиотек requests и pytest  
Сценарий: клиент создаёт заказ, затем получает данные о заказе по его трек-номеру и проверяет, что код ответа равен 200

## Чек-лист проверок
Автотест проверяет следующий сценарий:
1. Выполнить POST-запрос на создание заказа.
2. Сохранить номер трека заказа из ответа.
3. Выполнить GET-запрос на получение заказа по треку.
4. Проверить, что код ответа равен 200.
5. Убедиться, что трек заказа в ответе совпадает с сохранённым.

## Подготовка к запуску
1. Установите Python версии 3.8 или выше.
2. Установите зависимости командой:
pip install requests pytest

3. Убедитесь, что тестовый стенд доступен по адресу, указанному в файле test_order.py (переменная BASE_URL). Тестовый стенд:https://cca95327-803f-44ff-ad91-6f66141af256.serverhub.praktikum-services.ru/
4. Для запуска теста выполните команду:
python -m pytest tests/test_orders.py -v

## Структура запросов
- Создание заказа: POST /api/v1/orders 
- Тело запроса JSON с данными заказа.
- Получение заказа по треку: GET /api/v1/orders/track?t=<track>

## Примечания
- В проекте используется тестовый стенд, предоставленный для дипломной работы.
- Для корректной работы теста стенд должен быть активен и принимать запросы.
- Если тест падает с ошибкой, проверьте доступность сервера и актуальность эндпоинтов в документации API (/docs/).
